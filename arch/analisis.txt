## 🆕 SOLUSI PERGURUAN WORKFLOW

**Requirement**: Perguruan register → admin approve → user login sebagai **coach**.

**✅ JAWABAN**: **PAKAI FK SAJA** (`perguruan_id` di `users`). **TIDAK PERLU** tulis ulang nama perguruan di user table!

### Current Flow (Sudah Ada):
```
1. register-perguruan.blade.php → AuthController
   ↓ Buat User (status='pending', role='perguruan?')
   ↓ Buat Perguruan (user_id FK)

2. Admin/PerguruanController::verify($userId)
   ↓ Set user.status = 'active'
   ↓ Set user.perguruan_id = $perguruan->id
   ↓ **ADD**: $user->assignRole('coach')

3. Login → $user->perguruan->name (no duplication!)
```

### Edit Diperlukan:
**app/Http/Controllers/Admin/PerguruanController.php** (method `verify`):
```php
// Tambahkan ini setelah set status='active':
$user->assignRole('coach');  // Role Spatie
$user->fresh(); // Reload roles
```

**Akses Nama**: `$user->perguruan->name` di blade/controller.

**Test**:
```
1. POST /register-perguruan
2. Admin: PATCH /admin/perguruans/{id}/verify
3. Login → dashboard coach
```

**Mengapa FK?**
- ✅ Normalization (no duplication)

- ✅ Relasi Eloquent siap pakai

## 🆕 ERROR FIX PERGURUAN REGISTRATION

**Error**: `SQLSTATE[42703]: column "user_id" of relation "perguruans" does not exist`

**Cause**:
```
Migration 2026_04_01_035158: NO user_id column
AuthController::registerPerguruan(): INSERT dengan 'user_id'
```

**SOLUSI** (2 menit fix):

### 1. EDIT AuthController.php (registerPerguruan)
```php
DB::transaction(function () use ($user, $validated, $request, $logoPath) {
    // Buat Perguruan DULUAN (standalone)
    $perguruan = Perguruan::create([
        'name'    => $validated['perguruan_name'],
        'slug'    => Str::slug($validated['perguruan_name']),
        'address' => $validated['address'] ?? null,
        'phone'   => $validated['phone'] ?? null,
        'email'   => $validated['email'],
        'logo'    => $logoPath,
        // HAPUS 'user_id' ← tidak ada di migration!
    ]);

    // LINK ke User
    $user->update([
        'perguruan_id' => $perguruan->id,
    ]);

    // Assign role pending
    $user->assignRole('perguruan');
});
```

### 2. UPDATE Perguruan.php fillable
```php
protected $fillable = [
    'name', 'slug', 'address', 'phone', 'email', 'logo', 'is_active',
    // HAPUS 'user_id'
];
```

### 3. TEST
```
php artisan migrate:fresh --seed
php artisan serve
/register-perguruan → Success!
```

**Kenapa Design Ini Bagus?**
- 1 Perguruan → Many Users (coaches)
- User.status='pending' → Admin verify → role='coach'
- Normalization perfect ✅

**Result**: Register OK → Admin approve → Login dashboard coach! 🎉
